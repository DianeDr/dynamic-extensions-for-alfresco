# Contains the bean names of the Alfresco services and the service names under which 
# they are registered in OSGI space.
#
# Format: <bean name>[,<bean name>]:<service name>[,<servicename>...][:service type][:platform version]
#
# Note that for the purposes of using Alfresco host services from OSGI bundles, you 
# must use Java interfaces (i.e. "org.alfresco.service.cmr.repository.NodeService") 
# as OSGI service names. This ensures that autowiring OSGI bundle dependencies by 
# type works transparently. Furthermore, the bean must implement all the interfaces 
# specified. If it doesn't, it will not be registered as a service.
#
# It is possible to register multiple beans under a given service name.
# Distinguishing between multiple beans comes into play when client code declares
# a dependency to a specifically named bean using the @Named annotation.
#
# The service type property is used by the @AlfrescoService annotation to 
# distinguish between Alfresco services of the interface type.
# Valid values are "default" and "low-level"
#
# When registering multiple beans, the first bean is considered to be the default
# bean. (This bean is registered with the highest OSGi service ranking, so it 
# becomes the primary candidate when looking up services only by service name.)

# The list of services is based on public-services-context.xml from Alfresco 3.4.
